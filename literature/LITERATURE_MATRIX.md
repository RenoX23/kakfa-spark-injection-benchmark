# KSPFail — Literature Review Matrix

Synthesis matrix for the 20 papers in this collection. **Every number in the "Key results"
column was extracted from the actual paper PDF** (via `pypdf`, not from memory or abstracts) —
except the 3 survey papers marked `†` (paywalled, no local PDF; results described from the
verified abstract/record). Where a figure lives only in a plot I could not extract as text, I
say so rather than invent a value.

**Columns:** Paper · Problem addressed · Methodology · Dataset · Key results (real values) ·
Limitations · **Relevance & gap vs. KSPFail** (the last column is the value-add for our
related-work section — it says explicitly why each paper is *not* a competitor).

Grouped by the 5 collection themes. Full metadata + PDFs: see [`README.md`](README.md); BibTeX: [`references.bib`](references.bib).

## How to read the last column (for the paper's related-work section)

The gap argument is built by *category*, not paper-by-paper hand-waving:

1. **Method precedent exists** (Wu, Harrison, Lin, Time Machine) — ML failure prediction works; our contribution isn't "ML can predict failures."
2. **Fault-injection precedent exists** (Basiri, Natella, Chen-2025) — controlled injection is standard; ours isn't "injecting faults is novel."
3. **The Kafka/Spark systems-benchmark neighbour exists** (Vogel/DEBS'24) — but it measures *recovery speed across competing engines*, not *predictive lead time on a fixed pipeline*.
4. **Only one paper predicts lead time explicitly** (Time Machine) — different domain (HPC logs), no fault injection, no threshold baseline.

**The unoccupied intersection** = controlled fault injection **+** multi-model ML **+** lead-time evaluation **+** static-threshold baseline, on a **fixed Kafka + Spark Structured Streaming pipeline**. No paper in this matrix occupies all four. That is the gap.

- **Harrison's "88%" is recall, not accuracy.** Correct reading: overall accuracy 95.4%; failed-task precision 99% / recall 88%; successful-task precision 94%. (Earlier project notes had loosely called 88% an "accuracy on the failed-job class" — fixed here.)
- **Kafka (2011) throughput numbers** are reported only inside Figures 4–5; not extractable as text, so deliberately left unquoted rather than approximated.
- `†` = survey paper, no local PDF (paywalled ACM); its "results" are qualitative and taken from the verified abstract/record, not a downloaded full text.
